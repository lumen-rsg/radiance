# Radiance Agent

Система агента для терминала Radiance. Позволяет использовать LLM для кодогенерации проектов, редактирования файлов, поиска информации в Интернете, планировки задач и повседневного использования.

## Стек технологий
1. .NET 10 - Publish в Ready2Run. Позволяет закатить весь проект в один исполняемый файл с использованием лучшего от миров AOT и JIT
2. DaVinci - собственная система отрисовки в терминале. Использует подход React и components.
3. OpenRouter / OpenAI API - стандартный API для использования нейросетей.

## Основная идея
Система агентов представляет собой оркестрацию под-агентов:
1. Truffaldino - Агент оркестратор. Прямое общение с пользователем, вызов других агентов.
2. Lira - Агент поиска. (Explore Agent) - Сканирует кодовую базу. Предоставляет информацию о проекте.
3. Carmine - Агент планировки. Собирает план о имплементации, подготавливает информацию для агента имплементации.
4. Rhys - Агент имплементации. Агент, который вносит правки в код, выполняет команды, выполняет задачи.
5. Ava - агент поиска информации в Интернете с помощью утилит.
6. Vespera - легкий агент для вопросов (как side вопрос или /btw в Claude Code)

### Роли агентов (детально)

#### Truffaldino — Оркестратор
- Единая точка входа для всех запросов пользователя
- Анализирует запрос: нужен ли поиск по коду, планирование, редактирование, веб-поиск или просто ответ на вопрос
- Маршрутизация к под-агентам с передачей контекста
- Агрегирует результаты от нескольких под-агентов в единый ответ
- Управляет многошаговыми задачами (например: Lira ищет -> Carmine планирует -> Rhys реализует)
- Поддерживает общий контекст разговора между под-агентами
- **Модель**: Самая мощная из доступных (нужна для качественной маршрутизации)

#### Lira — Агент поиска (Explore Agent)
- Сканирует файловую систему и кодовую базу
- Ищет файлы по паттернам (glob), по содержимому (grep)
- Отвечает на вопросы о структуре проекта
- Предоставляет контекст другим агентам
- **Инструменты**: read_file, list_directory, run_command (только read-only: grep, find, cat)
- **Модель**: Быстрая, дешевая (задача простая, но требует много вызовов)

#### Carmine — Агент планировки
- Получает задачу от Truffaldino
- Собирает необходимый контекст через Lira или напрямую
- Разрабатывает пошаговый план имплементации
- Определяет файлы, которые нужно изменить
- Оценивает риски и зависимости
- Возвращает структурированный план в формате, понятном Rhys
- **Инструменты**: read_file, list_directory (для сбора контекста)
- **Модель**: Мощная (нужен анализ и архитектурное мышление)

#### Rhys — Агент имплементации
- Получает план от Carmine (через Truffaldino)
- Выполняет редактирование файлов (diff-based)
- Запускает команды для тестирования
- Итерирует до получения рабочего результата
- Откатывает изменения при неудаче
- **Инструменты**: read_file, edit_file, write_file, run_command
- **Модель**: Мощная с акцентом на code generation

#### Ava — Агент веб-поиска
- Ищет информацию в Интернете
- Читает и парсит веб-страницы
- Находит документацию, ответы на StackOverflow, актуальные API
- **Инструменты**: web_search, run_command (curl/wget)
- **Модель**: Средняя (поиск + суммаризация)

#### Vespera — Легкий агент Q&A
- Отвечает на быстрые вопросы без использования инструментов
- Боковые вопросы в процессе работы других агентов
- Аналог `/btw` в Claude Code
- **Инструменты**: Нет (только текстовая генерация)
- **Модель**: Быстрая, дешевая

## Инструменты агентов

### Реализованные

#### 1. run_command
Выполняет shell-команду через `/bin/bash -c`.
- Таймаут: 30 секунд
- Требует подтверждения пользователя (y/n)
- Возвращает stdout, stderr, exit code

#### 2. read_file
Читает содержимое файла с опциональным указанием диапазона строк.
- Лимит безопасности: 500 строк за запрос
- Возвращает содержимое с номерами строк

#### 3. write_file
Записывает файл целиком. Создаёт промежуточные директории при необходимости.
- Требует подтверждения пользователя (y/n)
- Перезаписывает существующие файлы

#### 4. list_directory
Листинг содержимого директории.
- Опциональная рекурсия
- Лимит: 200 записей

### Планируемые

#### 5. edit_file (Diff-based редактирование)
Вносит точечные правки в файл без перезаписи целиком.
- Формат: указание старого фрагмента и нового фрагмента
- Проверка уникальности старого фрагмента в файле
- Поддержка множественных правок за один вызов
- Автоматический откат при ошибке

#### 6. web_search
Поиск информации в Интернете.
- Интеграция с поисковым API (SearXNG, Tavily, или DuckDuckGo)
- Возвращает список релевантных результатов
- Опциональное чтение и суммаризация страниц

## Конфигурация

### agent.json (~/.radiance/agent.json)
```json
{
  "api_key": "sk-...",
  "base_url": "https://openrouter.ai/api/v1",
  "model": "anthropic/claude-sonnet-4-20250514",
  "system_prompt": "...",
  "max_tokens": 4096,
  "temperature": 0.7,
  "tools_enabled": true,
  "max_history": 50
}
```

### Переменные окружения (приоритет выше agent.json)
- `LIRA_API_KEY` / `OPENAI_API_KEY` — API ключ
- `LIRA_BASE_URL` / `OPENAI_BASE_URL` — Base URL
- `LIRA_MODEL` — Модель

### Поддерживаемые провайдеры
Любой OpenAI-совместимый API: OpenAI, OpenRouter, Ollama, LM Studio, vLLM, Together AI, Groq, и другие.

## Протокол взаимодействия агентов

### Формат контекста между агентами
```json
{
  "task_id": "uuid",
  "from_agent": "truffaldino",
  "to_agent": "carmine",
  "task": "описание задачи",
  "context": {
    "files": ["путь/к/файлу"],
    "search_results": [...],
    "constraints": [...]
  },
  "result": null
}
```

### Жизненный цикл многошаговой задачи
1. **Пользователь** → Truffaldino: "Добавь авторизацию в проект"
2. **Truffaldino** → Lira: "Сканируй проект, найди точки входа, middleware, модели"
3. **Lira** → Truffaldino: контекст проекта (структура, файлы, паттерны)
4. **Truffaldino** → Carmine: "Спланируй авторизацию на основе контекста"
5. **Carmine** → Truffaldino: пошаговый план (файлы, изменения, зависимости)
6. **Truffaldino** показывает план пользователю → подтверждение
7. **Truffaldino** → Rhys: "Реализуй план"
8. **Rhys** итеративно выполняет шаги, запускает тесты, исправляет ошибки
9. **Rhys** → Truffaldino: "Готово / Ошибка"
10. **Truffaldino** → Пользователь: итоговый результат

### Параллельное выполнение
Независимые под-задачи могут выполняться параллельно:
- Lira и Ava могут работать одновременно (поиск в коде + поиск в Интернете)
- Carmine всегда последовательно (ждёт контекст от Lira/Ava)
- Несколько задач Rhys могут идти параллельно для независимых файлов

## Безопасность и подтверждения

### Уровни опасности инструментов
| Уровень | Инструменты | Поведение |
|---------|------------|-----------|
| Безопасный | read_file, list_directory | Выполняется без подтверждения |
| Опасный | run_command, write_file | Требует y/n подтверждения |
| Критический | edit_file (системные файлы) | Требует явного подтверждения + предупреждение |

### Защита
- Sandbox для команд агента (ограничение рабочей директории)
- Лимит итераций tool loop (10 итераций)
- Лимит токенов на ответ
- Таймаут для команд (30 секунд)
- Запрет на destructive команды без подтверждения (rm -rf, git push --force, и т.д.)

## DaVinci — Система отрисовки

### Концепция
React-подобная система компонентов для терминала. Позволяет агентам и встроенным утилитам отрисовывать сложные интерфейсы в терминале.

### Принципы
1. **Компоненты** — независимые единицы UI с собственным состоянием и рендером
2. **Дерево компонентов** — иерархическая структура как в React
3. **Re-render** — перерисовка только изменившихся компонентов
4. **ANSI escape codes** — основа для позиционирования и цветов

### Планируемые компоненты
- `Text` — текст с цветом и стилями
- `Box` — прямоугольная рамка с контентом
- `List` — список с выделением
- `Table` — таблица с колонками
- `Spinner` — индикатор загрузки
- `ProgressBar` — прогресс-бар
- `CodeBlock` — блок кода с подсветкой синтаксиса
- `Markdown` — рендеринг Markdown в терминале
- `Input` — поле ввода
- `Selector` — интерактивный выбор из списка

### Интеграция с агентами
- Lira использует DaVinci для отображения результатов поиска
- Rhys использует DaVinci для показа diff'ов и прогресса
- Truffaldino использует DaVinci для интерактивных элементов (выбор плана, подтверждение)
